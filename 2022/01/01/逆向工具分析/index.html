实验题

[TOC]

0x01 前提

  • c 程序以及可执行的exe 文件

  • 调试程序: test.exe

  • 工具：ollydbg,IDAPro

  • 常见函数 strcmp功能:

    • ```
      int strcmp(const char *str1, const char *str2)
      str1 < str2 返回值 < 0
      str1 = str2 返回值 = 0
      str1 > str2 返回值 > 0

    • 发现栈帧中压入字符串1234567和字符串1234

      • 不难知道，0x42501c的地址存放的为字符串1234567，[arg.1]存放的为我们输入的字符串password1234。(猜测此时的1234567即为正确密码)，

    • 下面查看函数0x401260的功能。

      • 其实现了两个字符串之间逐字符的比较，因此应为strcmp函数
      • 二者比较后的结果存放在地址为[local.1]的空间中。

  • 回到main函数的栈帧中，cmp [local.1],0x0

    • 因为[local.1] 来自于 eax,eax的值为<0,所以明显cmp之后，标志位ZF仍为0，故不会跳转到0x4010f6

  • 然后压入字符串0x425068 空间内部的"incorrect password!"。

    • 作为参数传递给下面的printf函数，打印密码错误
    • 然后下面两步连续的跳转回到循环的开头,0x4010A5,重新开始新的循环。

  • 然后就轻车熟路，连续单步步过(F8)，一直到scanf函数的位置，输入我们猜测的密码1234567

    • 一路跳转到地址0x4010DE的位置，此时[loca.1] 对应eax存储的值为0。

    • 所以cmp [local.1],0x0 的结果中 ,ZF 值为1 ，故下面可以跳转。

    • 跳转到地址0x4010f6的区域，压入字符串"Congratulation! You have passed the verification!",并且调用打印函数。

    • 然后两步跳转跳到0x401107区域，退出循环，然后退出程序。

修改过程

  • 修改过程放在IDAPro 里面进行。
  • 首先直接查看该程序的整体框图。
    • 逻辑很清晰，在一个循环中，根据判断条件退出循环。
  • 因此我们只需要找到判断二者分支的判断语句，修改其逻辑即可以实现。
    • 观察框图，我们不难发现可以修改jz short loc_4010F6来实现逻辑翻转，即我们输入错误的密码一样可以输出Congratulation。
  • JZ 的机器指令为 74,我们只要将其修改为 jNZ，即可，JNZ的机器指令为75.
    • 然后调用将其保存。

实验结果

  • 修改前
    • 输入1234报错Incorrect，输入1234567输出conratulation,``
  • 修改后
    • 因为程序没有设置pause，程序运行成功后会闪退，因此我们在ollydbg上面进行调试，不难，发现我们这次输入错误的密码1234，终端可以输出congratulation,然而输入正确的1234567反而Incorrect

0x04 实验结构图

  • 修改前
  • 修改后，两张图，相比只修改了，判断分支的时候valid_flag是否等于0.

0x05 测试结论

  • 一个小小的分支语句的改变可以改变整个程序的逻辑，因此编写代码的时候一定要注意逻辑的顺序，一个小小的bug可能也会造成很大的损失。
  • 对于给出的数组password 数组的长度为1024，因为在main函数的栈帧里面，数组password是位于valid_flag上面的，因此原理上是可以实现通过栈溢出来覆盖valid_flag的值来达到拿到shell的目的。

附加题

[TOC]

0x01 前提

  • crackme.exe

  • 工具：

    • IDAPro
    • ollydbg

  • 汇编指令对应的机器码

    • ```
      JZ => 74
      JNZ => 75

      • sbb是带借位减法指令，它利用了CF位上记录的借位值。

        功能:操作对象1=操作对象1-操作对象2-CF

      • 对于该段函数，结合上面的程序，显然上述的cf为0，

      • 故这里的 是将 eax 置为 0x1

    • 上述的eax 到这里，如果密码正确，eax = 0,test eax,eax产生的ZF 为0，所以跳转到相反的地址,弹框注册成功。
    • 如果密码错误，eax = 1 ,test eax,eax 产生的 ZF 为 1， 跳转到函数loc_401637，弹框注册失败，然后继续输入数据，

修改过程

  • 根据逻辑，我们要想实现输入错误的密码也可以弹框注册成功
  • 这里提供两个策略
    • 一 修改 jnz loc_401637的逻辑，实现输入错误的密码会跳转到注册成功，反而输入正确的密码crackmepassword反而会弹框注册失败
    • 二 修改到达函数loc_401611的路径，使比较字符错误后不会跳到该函数，这样就实现了输入错误密码，最后跳转到函数loc_40160D，最后成功输出注册成功，输入正确密码反而会弹出注册失败。
  • 法一 ： 修改 jnz loc_401637
    • 将这里修改为jz loc_401637
  • 法二： 修改达到函数loc_401611的分支
    • 这里需要修改两个地方的逻辑
    • 即将jnz loc_401611均改为jz loc_401611
    • 修改后

实验结果

  • 未修改前
    • 输入正确密码 crackmepassword
    • 输入错误密码 12345
  • 修改方案一 修改 jnz loc_401637
    • 输入正确密码 crackmepassword
    • 输入错误密码 12345
  • 修改方案二 修改达到函数loc_401611的分支
    • 输入正确密码 crackmepassword
    • 输入错误密码

0x05 测试结论

  • 对于比较复杂的逻辑，直观的观察流程图更为方便。
  • 对于某些不太清晰的数据流，可以通过olldbg来进行动态调试来观察数据的走向来确定数据的流向，进而确定程序的逻辑结构。
